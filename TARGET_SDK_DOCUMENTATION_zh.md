# 应用的目标 SDK 文档

[简体中文] | [English](TARGET_SDK_DOCUMENTATION.md)

这是 `targetSdk` 的文档，旨在确保与各种新型手机的兼容性。

目前，中国大陆要求的目标 SDK 为 30 或更高（未来要求为 33 或更高）。

目前，Google Play 上大多数应用的目标 SDK 为 35 或更高（要求每年更新一次）。

本应用每年升级一次其目标 SDK 和编译 SDK。

## 1. 降级到版本 36 的影响
   - 影响：系统不再强制执行 Android 17 中引入的新限制（如无锁 MessageQueue 实现、动态加载的 .so 文件必须为只读、以及短信验证码延迟）。
   - 老年用户体验：它将不受 Android 17 中引入的新限制的影响（如大屏幕的强制方向锁定和有关局域网设备发现的保护），但也将错过相应的安全增强功能。
   - 风险：如果设备运行 Android 17 或更高版本且 `targetSdk=36`，系统将遵循 Android 16 的行为，可能导致设备上缺失新的系统功能。
## 2. 降级到版本 35 的影响
   - 影响：
     - 默认不强制执行全屏布局；状态栏或导航栏可能会遮挡您的 Compose 布局。
     - 预测性返回动画未启用；返回手势仍遵循旧逻辑。
     - dataSync 后台任务的超时策略按旧版本处理。
   - 老年用户体验：在桌面 UI 上，状态栏可能会重叠（与之前提到的 ClockWidget 定位问题有关）；您需要手动处理 `WindowInsets`，否则在 Android 15+ 设备上显示将出现异常。
## 3. 降级到版本 34 的影响
   - 影响：
     - 前台服务必须显式声明 `foregroundServiceType`；否则，较新的系统版本可能会抛出异常（但如果 `targetSdkVersion` 低于 34，系统目前不强制执行此操作）。
     - 精准闹钟的权限策略已放宽，以匹配旧行为。
   - 老年用户体验：天气更新和闹钟提醒的后台进程在运行时受到的限制较少，但也更容易被系统终止，可能导致老年用户错过天气警告。
## 4. 降级到版本 33 的影响
   - 影响：
     - 通知权限不是强制性的，因此系统不会提示用户允许通知；因此，老年用户可能无法收到天气预警 Snackbar。
     - 媒体权限已细化；存储/媒体访问遵循旧逻辑。
   - 老年用户体验：天气警报和来电通知可能会静默失败，因为它们不遵循新的通知权限流程。
## 5. 降级到版本 31 的影响
   - 影响：
     - 不强制执行从后台启动 Activity 的限制，允许应用在后台运行时弹出界面（这是一个容易被滥用的功能，尽管对于意外触发动作的老年用户来说可能确实有用）。
     - 通知必须使用标准模板；对自定义通知区域没有限制（仅针对目标 API 级别 31 或更高版本强制要求模板）。
     - 通过蓝牙权限拆分的细粒度控制不生效。
     - SplashScreen 行为遵循旧实现。
   - 风险：自定义通知布局在较新的系统版本上可能显示不正确，但当 `targetSdkVersion` 为 31 或更低时，系统会保持与旧样式的兼容性。
## 6. 降级到版本 30 的影响
   - 影响：
     - 对软件包可见性没有限制；您的启动器可以检测到所有已安装的应用，而无需声明 `<queries>`。
     - 在 Android 10 设备上不完全强制执行分区存储（当 `targetSdkVersion` 为 29 时，可以通过设置 `requestLegacyExternalStorage` 来绕过）。
   - 老年用户体验：依赖旧的存储模型进行天气缓存和相册访问会更简单，但这也带来了更大的隐私风险（因为应用可以遍历外部存储）。

根据长辈桌面的要求，默认目标 SDK 设置为 37，除非特定要求另有规定。
